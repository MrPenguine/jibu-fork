## 1. INTRODUCTION TO CONTROLLERS

Controllers in the Jibu Console backend are responsible for handling HTTP requests and defining API endpoints. They act as the entry point for client requests and delegate the business logic to services. Controllers are implemented using NestJS decorators and follow RESTful API design principles.

The primary responsibilities of controllers include:
- Defining API routes and HTTP methods
- Validating input data using DTOs and pipes
- Delegating business logic to services
- Formatting and returning responses
- Handling request-specific error handling
- Implementing authentication and authorization checks

## 2. CONTROLLER IMPLEMENTATION PATTERN

Controllers in Jibu Console follow a consistent implementation pattern:

1. Class Declaration with @Controller Decorator:
```typescript
@Controller('v1/resource-name')
export class ResourceController {
  constructor(private readonly resourceService: ResourceService) {}
}
```

2. Endpoint Definition with HTTP Method Decorators:
```typescript
@Get()
findAll(@Query() query: PaginationQueryDto): Promise<PaginatedResponse<Resource>> {
  return this.resourceService.findAll(query);
}

@Get(':id')
findOne(@Param('id') id: string): Promise<Resource> {
  return this.resourceService.findOne(id);
}

@Post()
create(@Body() createResourceDto: CreateResourceDto): Promise<Resource> {
  return this.resourceService.create(createResourceDto);
}

@Put(':id')
update(
  @Param('id') id: string,
  @Body() updateResourceDto: UpdateResourceDto,
): Promise<Resource> {
  return this.resourceService.update(id, updateResourceDto);
}

@Delete(':id')
remove(@Param('id') id: string): Promise<void> {
  return this.resourceService.remove(id);
}
```

3. Request Validation using DTOs and Pipes:
```typescript
@Post()
@UsePipes(new ValidationPipe({ transform: true }))
create(@Body() createResourceDto: CreateResourceDto): Promise<Resource> {
  return this.resourceService.create(createResourceDto);
}
```

4. Authentication and Authorization using Guards:
```typescript
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles(Role.ADMIN)
@Delete(':id')
remove(@Param('id') id: string): Promise<void> {
  return this.resourceService.remove(id);
}
```

5. Response Transformation using Interceptors:
```typescript
@UseInterceptors(ClassSerializerInterceptor)
@Get(':id')
findOne(@Param('id') id: string): Promise<Resource> {
  return this.resourceService.findOne(id);
}
```

## 4. REQUEST VALIDATION

Controllers in Jibu Console use Data Transfer Objects (DTOs) and validation pipes to ensure that incoming requests contain valid data. This validation is implemented using class-validator and class-transformer libraries.

Example DTO with validation:
```typescript
export class CreateUserDto {
  @IsString()
  @IsNotEmpty()
  @MinLength(2)
  @MaxLength(50)
  name: string;

  @IsEmail()
  @IsNotEmpty()
  email: string;

  @IsString()
  @IsNotEmpty()
  @MinLength(8)
  @Matches(/((?=.*\d)|(?=.*\W+))(?![.\n])(?=.*[A-Z])(?=.*[a-z]).*$/, {
    message: 'Password is too weak',
  })
  password: string;

  @IsOptional()
  @IsString()
  @IsUUID()
  organizationId?: string;
}
```

Validation is applied using the ValidationPipe:
```typescript
@Post()
@UsePipes(new ValidationPipe({ transform: true }))
create(@Body() createUserDto: CreateUserDto): Promise<User> {
  return this.userService.create(createUserDto);
}
```

## 6. ERROR HANDLING

Controllers in Jibu Console implement error handling using NestJS exception filters and custom exceptions. This ensures that errors are caught and formatted consistently.

1. Custom exceptions:
```typescript
export class ResourceNotFoundException extends HttpException {
  constructor(resourceName: string, id: string) {
    super(`${resourceName} with ID ${id} not found`, HttpStatus.NOT_FOUND);
  }
}
```

2. Exception filters:
```typescript
@Catch(HttpException)
export class HttpExceptionFilter implements ExceptionFilter {
  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse<Response>();
    const request = ctx.getRequest<Request>();
    const status = exception.getStatus();

    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      path: request.url,
      message: exception.message,
    });
  }
}
```

3. Application of exception filters:
```typescript
@UseFilters(new HttpExceptionFilter())
@Controller('v1/user')
export class UserController {
  // ...
}
```

## 7. AUTHENTICATION AND AUTHORIZATION

Controllers in Jibu Console implement authentication and authorization using guards and decorators. This ensures that only authorized users can access protected endpoints.

1. Authentication guard:
```typescript
@Injectable()
export class JwtAuthGuard implements CanActivate {
  constructor(private readonly reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean | Promise<boolean> | Observable<boolean> {
    // Authentication logic
  }
}
```

2. Role-based authorization:
```typescript
@SetMetadata('roles', roles)
export const Roles = (...roles: Role[]) => SetMetadata('roles', roles);

@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const roles = this.reflector.get<Role[]>('roles', context.getHandler());
    if (!roles) {
      return true;
    }
    const request = context.switchToHttp().getRequest();
    const user = request.user;
    return this.matchRoles(roles, user.roles);
  }

  private matchRoles(roles: Role[], userRoles: Role[]): boolean {
    return roles.some(role => userRoles.includes(role));
  }
}
```

3. Application of guards:
```typescript
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles(Role.ADMIN)
@Delete(':id')
remove(@Param('id') id: string): Promise<void> {
  return this.userService.remove(id);
}
```

The controllers in Jibu Console implement a comprehensive authentication and authorization system that ensures secure access to API endpoints while providing flexibility for different user roles and permissions.
