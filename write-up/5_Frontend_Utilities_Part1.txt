1. INTRODUCTION TO FRONTEND UTILITIES
==============================================================

The frontend utilities in Jibu Console provide a set of tools and abstractions that simplify common tasks, promote code reuse, and ensure consistency across the application. These utilities include API clients for interacting with the backend, context providers for state management, authentication utilities, and various helper functions.

The frontend utilities are organized into several categories:

1. API Utilities: Clients for interacting with the backend APIs
2. Context Providers: React contexts for global state management
3. Authentication Utilities: Utilities for handling user authentication
4. Helper Functions: General-purpose utility functions

This document focuses on the API utilities, which provide type-safe clients for interacting with the backend APIs.

==============================================================
2. API UTILITIES
==============================================================

The API utilities in Jibu Console provide type-safe clients for interacting with the backend APIs. They encapsulate the details of API requests and responses, handle error handling, and provide a consistent interface for all API operations.

2.1. Base API Client
------------------

The base API client provides common functionality for all API clients, such as request handling, error handling, and authentication.

File: apps/frontend/src/utils/api.ts

The base API client is implemented as a class that provides methods for making HTTP requests to the backend. It handles authentication, error handling, and response parsing.

Key Features:
- Type-safe request and response handling
- Automatic authentication token management
- Consistent error handling
- Support for pagination and filtering
- Request caching and deduplication

Implementation:

```typescript
export class ApiClient {
  private baseUrl: string;
  private headers: Record<string, string>;
  private cache: Map<string, { data: any; timestamp: number }> = new Map();
  private cacheTTL: number = 5000; // 5 seconds

  constructor(baseUrl: string, headers: Record<string, string> = {}) {
    this.baseUrl = baseUrl;
    this.headers = {
      'Content-Type': 'application/json',
      ...headers,
    };
  }

  async get<T>(
    endpoint: string,
    params?: Record<string, any>,
    options?: RequestOptions,
  ): Promise<T> {
    const url = this.buildUrl(endpoint, params);
    
    // Check cache if caching is enabled
    if (options?.cache && !options?.skipCache) {
      const cachedData = this.getCachedData<T>(url);
      if (cachedData) {
        return cachedData;
      }
    }

    const response = await fetch(url, {
      method: 'GET',
      headers: this.getHeaders(options?.headers),
      credentials: 'include',
    });

    const data = await this.handleResponse<T>(response);
    
    // Cache data if caching is enabled
    if (options?.cache) {
      this.cacheData(url, data);
    }

    return data;
  }

  async post<T>(
    endpoint: string,
    body?: any,
    options?: RequestOptions,
  ): Promise<T> {
    const url = this.buildUrl(endpoint);

    const response = await fetch(url, {
      method: 'POST',
      headers: this.getHeaders(options?.headers),
      credentials: 'include',
      body: body ? JSON.stringify(body) : undefined,
    });

    return this.handleResponse<T>(response);
  }

  async put<T>(
    endpoint: string,
    body?: any,
    options?: RequestOptions,
  ): Promise<T> {
    const url = this.buildUrl(endpoint);

    const response = await fetch(url, {
      method: 'PUT',
      headers: this.getHeaders(options?.headers),
      credentials: 'include',
      body: body ? JSON.stringify(body) : undefined,
    });

    return this.handleResponse<T>(response);
  }

  async delete<T>(
    endpoint: string,
    options?: RequestOptions,
  ): Promise<T> {
    const url = this.buildUrl(endpoint);

    const response = await fetch(url, {
      method: 'DELETE',
      headers: this.getHeaders(options?.headers),
      credentials: 'include',
    });

    return this.handleResponse<T>(response);
  }

  private buildUrl(endpoint: string, params?: Record<string, any>): string {
    const url = new URL(`${this.baseUrl}${endpoint}`);
    
    if (params) {
      Object.entries(params).forEach(([key, value]) => {
        if (value !== undefined && value !== null) {
          if (typeof value === 'object') {
            url.searchParams.append(key, JSON.stringify(value));
          } else {
            url.searchParams.append(key, String(value));
          }
        }
      });
    }
    
    return url.toString();
  }

  private getHeaders(additionalHeaders?: Record<string, string>): Record<string, string> {
    return {
      ...this.headers,
      ...additionalHeaders,
    };
  }

  private async handleResponse<T>(response: Response): Promise<T> {
    if (!response.ok) {
      const errorData = await response.json().catch(() => ({}));
      throw new ApiError(
        errorData.message || 'An error occurred',
        response.status,
        errorData,
      );
    }

    // Handle empty responses
    if (response.status === 204) {
      return {} as T;
    }

    return response.json();
  }

  private getCachedData<T>(key: string): T | null {
    const cachedItem = this.cache.get(key);
    
    if (cachedItem && Date.now() - cachedItem.timestamp < this.cacheTTL) {
      return cachedItem.data as T;
    }
    
    return null;
  }

  private cacheData(key: string, data: any): void {
    this.cache.set(key, {
      data,
      timestamp: Date.now(),
    });
  }

  clearCache(): void {
    this.cache.clear();
  }

  invalidateCache(endpoint: string): void {
    const prefix = this.buildUrl(endpoint);
    
    for (const key of this.cache.keys()) {
      if (key.startsWith(prefix)) {
        this.cache.delete(key);
      }
    }
  }
}

export class ApiError extends Error {
  status: number;
  data: any;

  constructor(message: string, status: number, data: any) {
    super(message);
    this.name = 'ApiError';
    this.status = status;
    this.data = data;
  }
}

export interface RequestOptions {
  headers?: Record<string, string>;
  cache?: boolean;
  skipCache?: boolean;
}

export interface PaginationParams {
  page?: number;
  limit?: number;
  filter?: Record<string, any>;
  sort?: Record<string, 'ASC' | 'DESC'>;
}

export interface PaginatedResponse<T> {
  items: T[];
  meta: {
    totalItems: number;
    itemCount: number;
    itemsPerPage: number;
    totalPages: number;
    currentPage: number;
  };
}
```

2.2. Agent API Client
-------------------

The Agent API client provides methods for interacting with the agent-related endpoints of the backend API.

File: apps/frontend/src/utils/AgentApi.ts

The Agent API client extends the base API client and provides methods for creating, retrieving, updating, and deleting agents, as well as executing agents.

Key Methods:
- getAgents: Retrieves a list of agents with pagination and filtering
- getAgent: Retrieves a single agent by ID
- createAgent: Creates a new agent
- updateAgent: Updates an existing agent
- deleteAgent: Deletes an agent
- executeAgent: Executes an agent with input

Implementation:

```typescript
import { ApiClient, PaginatedResponse, PaginationParams } from './api';

export interface Agent {
  id: string;
  name: string;
  description: string;
  organizationId: string;
  config: AgentConfig;
  knowledgeBaseIds: string[];
  credentialId?: string;
  isPublic: boolean;
  createdAt: string;
  updatedAt: string;
}

export interface AgentConfig {
  provider: string;
  model: string;
  temperature: number;
  maxTokens?: number;
  topP?: number;
  frequencyPenalty?: number;
  presencePenalty?: number;
  stopSequences?: string[];
  systemPrompt?: string;
  tools?: AgentTool[];
  options?: Record<string, any>;
}

export interface AgentTool {
  type: string;
  name: string;
  description: string;
  config?: Record<string, any>;
}

export interface CreateAgentDto {
  name: string;
  description: string;
  organizationId: string;
  config: AgentConfig;
  knowledgeBaseIds?: string[];
  credentialId?: string;
  isPublic?: boolean;
}

export interface UpdateAgentDto {
  name?: string;
  description?: string;
  config?: AgentConfig;
  knowledgeBaseIds?: string[];
  credentialId?: string;
  isPublic?: boolean;
}

export interface ExecuteAgentDto {
  input: string;
  options?: Record<string, any>;
}

export interface AgentExecutionResult {
  output: string;
  metadata?: Record<string, any>;
}

export class AgentApi {
  private apiClient: ApiClient;

  constructor(apiClient: ApiClient) {
    this.apiClient = apiClient;
  }

  async getAgents(params?: PaginationParams): Promise<PaginatedResponse<Agent>> {
    return this.apiClient.get<PaginatedResponse<Agent>>('/v1/agent', params, { cache: true });
  }

  async getAgent(id: string): Promise<Agent> {
    return this.apiClient.get<Agent>(`/v1/agent/${id}`, undefined, { cache: true });
  }

  async createAgent(createAgentDto: CreateAgentDto): Promise<Agent> {
    const agent = await this.apiClient.post<Agent>('/v1/agent', createAgentDto);
    this.apiClient.invalidateCache('/v1/agent');
    return agent;
  }

  async updateAgent(id: string, updateAgentDto: UpdateAgentDto): Promise<Agent> {
    const agent = await this.apiClient.put<Agent>(`/v1/agent/${id}`, updateAgentDto);
    this.apiClient.invalidateCache('/v1/agent');
    this.apiClient.invalidateCache(`/v1/agent/${id}`);
    return agent;
  }

  async deleteAgent(id: string): Promise<void> {
    await this.apiClient.delete<void>(`/v1/agent/${id}`);
    this.apiClient.invalidateCache('/v1/agent');
    this.apiClient.invalidateCache(`/v1/agent/${id}`);
  }

  async executeAgent(id: string, executeAgentDto: ExecuteAgentDto): Promise<AgentExecutionResult> {
    return this.apiClient.post<AgentExecutionResult>(`/v1/agent/${id}/execute`, executeAgentDto);
  }
}
```

2.3. Assistants API Client
------------------------

The Assistants API client provides methods for interacting with the assistant-related endpoints of the backend API.

File: apps/frontend/src/utils/AssistantsApi.ts

The Assistants API client extends the base API client and provides methods for creating, retrieving, updating, and deleting assistants, as well as deploying assistants and checking deployment status.

Key Methods:
- getAssistants: Retrieves a list of assistants with pagination and filtering
- getAssistant: Retrieves a single assistant by ID
- createAssistant: Creates a new assistant
- updateAssistant: Updates an existing assistant
- deleteAssistant: Deletes an assistant
- deployAssistant: Deploys an assistant
- getDeploymentStatus: Checks the deployment status of an assistant

Implementation:

