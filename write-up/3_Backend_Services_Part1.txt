1. INTRODUCTION TO SERVICES
==============================================================

Services in the Jibu Console backend are responsible for implementing the business logic of the application. They act as an intermediary between controllers and data access layers, encapsulating complex operations and ensuring data consistency. Services are implemented using NestJS's dependency injection system, making them highly testable and maintainable.

The primary responsibilities of services include:
- Implementing business logic and domain rules
- Coordinating between multiple repositories or other services
- Handling domain-specific validation
- Managing transactions and ensuring data consistency
- Implementing caching strategies where appropriate
- Integrating with external systems and APIs

==============================================================
2. SERVICE IMPLEMENTATION PATTERN
==============================================================

Services in Jibu Console follow a consistent implementation pattern:

1. Class Declaration with @Injectable Decorator:
```typescript
@Injectable()
export class ResourceService {
  constructor(
    @InjectRepository(ResourceEntity)
    private readonly resourceRepository: Repository<ResourceEntity>,
    private readonly otherService: OtherService,
    private readonly configService: ConfigService,
  ) {}
}
```

2. CRUD Operations:
```typescript
async create(createResourceDto: CreateResourceDto): Promise<Resource> {
  const resource = this.resourceRepository.create(createResourceDto);
  return this.resourceRepository.save(resource);
}

async findAll(options: PaginationQueryDto): Promise<PaginatedResponse<Resource>> {
  const [items, totalItems] = await this.resourceRepository.findAndCount({
    skip: (options.page - 1) * options.limit,
    take: options.limit,
    where: options.filter,
    order: options.sort,
  });

  return {
    items,
    meta: {
      totalItems,
      itemCount: items.length,
      itemsPerPage: options.limit,
      totalPages: Math.ceil(totalItems / options.limit),
      currentPage: options.page,
    },
  };
}

async findOne(id: string): Promise<Resource> {
  const resource = await this.resourceRepository.findOne({ where: { id } });
  if (!resource) {
    throw new ResourceNotFoundException('Resource', id);
  }
  return resource;
}

async update(id: string, updateResourceDto: UpdateResourceDto): Promise<Resource> {
  const resource = await this.findOne(id);
  const updatedResource = this.resourceRepository.merge(resource, updateResourceDto);
  return this.resourceRepository.save(updatedResource);
}

async remove(id: string): Promise<void> {
  const resource = await this.findOne(id);
  await this.resourceRepository.remove(resource);
}
```

3.1. API Key Service
------------------

The API Key Service manages the creation, retrieval, updating, and deletion of API keys. API keys are used for authenticating API requests from external systems.

File: apps/backend/src/modules/v1/api-key/api-key.service.ts

Key Methods:
- createApiKey(createApiKeyDto): Creates a new API key
- findAll(options): Retrieves all API keys with pagination and filtering
- findOne(id): Retrieves a single API key by ID
- update(id, updateApiKeyDto): Updates an API key's metadata
- remove(id): Deletes an API key
- validateApiKey(key): Validates an API key and returns the associated user and organization
- rotateApiKey(id): Generates a new API key value while preserving metadata

Implementation Details:
- Uses TypeORM for data access
- Implements secure generation and storage of API keys
- Supports filtering by organization ID
- Implements caching for frequently accessed API keys
- Handles API key rotation with proper invalidation of old keys
- Integrates with the user and organization services for validation

Code Example (Simplified):
```typescript
@Injectable()
export class ApiKeyService {
  constructor(
    @InjectRepository(ApiKeyEntity)
    private readonly apiKeyRepository: Repository<ApiKeyEntity>,
    private readonly userService: UserService,
    private readonly organizationService: OrganizationService,
  ) {}

  async createApiKey(createApiKeyDto: CreateApiKeyDto): Promise<ApiKey> {
    // Validate user and organization
    await this.userService.findOne(createApiKeyDto.userId);
    await this.organizationService.findOne(createApiKeyDto.organizationId);

    // Generate API key
    const keyValue = this.generateApiKey();
    const hashedKey = await this.hashApiKey(keyValue);

    // Create API key entity
    const apiKey = this.apiKeyRepository.create({
      ...createApiKeyDto,
      key: hashedKey,
    });

    // Save API key
    const savedApiKey = await this.apiKeyRepository.save(apiKey);

    // Return API key with plain text value (only shown once)
    return {
      ...savedApiKey,
      key: keyValue,
    };
  }

  async validateApiKey(key: string): Promise<{ user: User; organization: Organization }> {
    const hashedKey = await this.hashApiKey(key);
    
    const apiKey = await this.apiKeyRepository.findOne({
      where: { key: hashedKey, isActive: true },
      relations: ['user', 'organization'],
    });

    if (!apiKey) {
      throw new UnauthorizedException('Invalid API key');
    }

    if (apiKey.expiresAt && apiKey.expiresAt < new Date()) {
      throw new UnauthorizedException('API key has expired');
    }

    return {
      user: apiKey.user,
      organization: apiKey.organization,
    };
  }

  private generateApiKey(): string {
    // Generate a secure random API key
    return randomBytes(32).toString('hex');
  }

  private async hashApiKey(key: string): Promise<string> {
    // Hash API key for secure storage
    return createHash('sha256').update(key).digest('hex');
  }
}
```

3.2. Assistant Service
--------------------

The Assistant Service manages the creation, retrieval, updating, and deletion of assistants. Assistants combine agents with knowledge bases and other settings to create user-facing AI assistants.

File: apps/backend/src/modules/v1/assistants/assistant.service.ts

Key Methods:
- createAssistant(createAssistantDto): Creates a new assistant
- findAll(options): Retrieves all assistants with pagination and filtering
- findOne(id): Retrieves a single assistant by ID
- update(id, updateAssistantDto): Updates an assistant's configuration
- remove(id): Deletes an assistant
- deployAssistant(id): Deploys an assistant to make it available for use
- getDeploymentStatus(id): Checks the deployment status of an assistant
- assignKnowledgeBase(id, knowledgeBaseId): Assigns a knowledge base to an assistant
- removeKnowledgeBase(id, knowledgeBaseId): Removes a knowledge base from an assistant

Implementation Details:
- Uses prisma for data access
- Supports filtering by organization ID
- Implements caching for frequently accessed assistants
- Handles assistant deployment and status checking
- Integrates with the agent and knowledge base services
- Manages assistant configurations and settings

Code Example (Simplified):
```typescript
@Injectable()
export class AssistantService {
  constructor(
    @InjectRepository(AssistantEntity)
    private readonly assistantRepository: Repository<AssistantEntity>,
    private readonly agentService: AgentService,
    private readonly knowledgeBaseService: KnowledgeBaseService,
    private readonly deploymentService: DeploymentService,
  ) {}

  async createAssistant(createAssistantDto: CreateAssistantDto): Promise<Assistant> {
    // Validate agent
    await this.agentService.findOne(createAssistantDto.agentId);

    // Validate knowledge bases if provided
    if (createAssistantDto.knowledgeBaseIds) {
      for (const knowledgeBaseId of createAssistantDto.knowledgeBaseIds) {
        await this.knowledgeBaseService.findOne(knowledgeBaseId);
      }
    }

    // Create assistant entity
    const assistant = this.assistantRepository.create(createAssistantDto);

    // Save assistant
    return this.assistantRepository.save(assistant);
  }

  async deployAssistant(id: string): Promise<DeploymentStatus> {
    const assistant = await this.findOne(id);

    // Check if assistant is already deployed
    if (assistant.deploymentStatus === 'deployed') {
      return { status: 'deployed', assistantId: id };
    }

    // Update deployment status
    assistant.deploymentStatus = 'deploying';
    await this.assistantRepository.save(assistant);

    // Trigger deployment process
    const deploymentResult = await this.deploymentService.deployAssistant(assistant);

    // Update assistant with deployment result
    assistant.deploymentStatus = deploymentResult.status;
    assistant.deploymentId = deploymentResult.deploymentId;
    await this.assistantRepository.save(assistant);

    return {
      status: deploymentResult.status,
      assistantId: id,
      deploymentId: deploymentResult.deploymentId,
    };
  }

  async assignKnowledgeBase(id: string, knowledgeBaseId: string): Promise<Assistant> {
    const assistant = await this.findOne(id);
    const knowledgeBase = await this.knowledgeBaseService.findOne(knowledgeBaseId);

    // Check if knowledge base is already assigned
    if (assistant.knowledgeBases.some(kb => kb.id === knowledgeBaseId)) {
      return assistant;
    }

    // Assign knowledge base
    assistant.knowledgeBases.push(knowledgeBase);
    return this.assistantRepository.save(assistant);
  }
}
```

3.3. Chats Service
----------------

The Chats Service manages chat sessions and message history. It provides methods for creating chat sessions, sending messages, and retrieving chat history.

File: apps/backend/src/modules/v1/chats/chats.service.ts

Key Methods:
- createChat(createChatDto): Creates a new chat session
- findAll(options): Retrieves all chats with pagination and filtering
- findOne(id): Retrieves a single chat by ID
- update(id, updateChatDto): Updates a chat's metadata
- remove(id): Deletes a chat
- sendMessage(id, messageDto): Sends a message in a chat
- getMessages(id, options): Retrieves messages in a chat
- streamMessages(id): Streams messages in a chat in real-time

Implementation Details:
- Uses prisma for data access
- Implements Redis-based chat history storage
- Supports filtering by organization ID and assistant ID
- Implements caching for frequently accessed chats
- Handles real-time message streaming using Server-Sent Events (SSE)
- Integrates with the assistant service for message processing

The Chats Service implements a Redis-based chat history system that integrates with Langflow workflows. The system stores chat messages in Redis under two key patterns:
1. `chat:history:{chatId}` - Direct access by chat ID
2. `chat:{assistantId}:{sessionId}` - Backward compatibility

It includes a local caching mechanism with a 5-second TTL for performance optimization and supports knowledge base integration using the `kb_{knowledgeBaseId}` collection naming pattern.

Code Example (Simplified):
```typescript
@Injectable()
export class ChatsService {
  private readonly logger = new Logger(ChatsService.name);
  private readonly localCache: Map<string, { messages: ChatMessage[]; timestamp: number }> = new Map();
  private readonly cacheTTL = 5000; // 5 seconds

  constructor(
    @InjectRepository(ChatEntity)
    private readonly chatRepository: Repository<ChatEntity>,
    @InjectRedis() private readonly redis: Redis,
    private readonly assistantService: AssistantService,
    private readonly langflowChatHistoryService: LangflowChatHistoryService,
  ) {}

  async createChat(createChatDto: CreateChatDto): Promise<Chat> {
    // Validate assistant
    await this.assistantService.findOne(createChatDto.assistantId);

    // Create chat entity
    const chat = this.chatRepository.create(createChatDto);

    // Save chat
    return this.chatRepository.save(chat);
  }

  async sendMessage(id: string, messageDto: SendMessageDto): Promise<ChatMessage> {
    const chat = await this.findOne(id);

    // Create message
    const message: ChatMessage = {
      id: uuidv4(),
      chatId: id,
      content: messageDto.content,
      role: messageDto.role,
      createdAt: new Date().toISOString(),
    };

    // Store message in Redis
    await this.storeMessage(id, message);

    // If user message, process with assistant
    if (messageDto.role === 'user') {
      // Process message with assistant (async)
      this.processMessageWithAssistant(chat, message).catch(error => {
        this.logger.error(`Error processing message with assistant: ${error.message}`, error.stack);
      });
    }

    return message;
  }

  async getMessages(id: string, options?: { limit?: number; before?: string }): Promise<ChatMessage[]> {
    const chat = await this.findOne(id);

    // Check local cache first
    const cacheKey = `chat:${id}:messages`;
    const cachedData = this.localCache.get(cacheKey);
    
    if (cachedData && Date.now() - cachedData.timestamp < this.cacheTTL) {
      return this.filterMessages(cachedData.messages, options);
    }

    // Get messages from Redis
    const messages = await this.langflowChatHistoryService.getChatHistory(id);

    // Update local cache
    this.localCache.set(cacheKey, {
      messages,
      timestamp: Date.now(),
    });

    return this.filterMessages(messages, options);
  }

  private async storeMessage(chatId: string, message: ChatMessage): Promise<void> {
    // Store in Redis using both key patterns
    await this.redis.rpush(`chat:history:${chatId}`, JSON.stringify(message));
    
    // Get chat for assistant and session IDs
    const chat = await this.findOne(chatId);
    await this.redis.rpush(`chat:${chat.assistantId}:${chat.sessionId}`, JSON.stringify(message));

    // Update local cache
    const cacheKey = `chat:${chatId}:messages`;
    const cachedData = this.localCache.get(cacheKey);
    
    if (cachedData) {
      cachedData.messages.push(message);
      cachedData.timestamp = Date.now();
    }
  }

  private async processMessageWithAssistant(chat: Chat, userMessage: ChatMessage): Promise<void> {
    // Get assistant
    const assistant = await this.assistantService.findOne(chat.assistantId);

    // Get chat history
    const history = await this.getMessages(chat.id);

    // Process message with assistant
    const response = await this.assistantService.processMessage(
      assistant,
      userMessage.content,
      history,
    );

    // Create assistant message
    const assistantMessage: ChatMessage = {
      id: uuidv4(),
      chatId: chat.id,
      content: response.content,
      role: 'assistant',
      createdAt: new Date().toISOString(),
    };

    // Store assistant message
    await this.storeMessage(chat.id, assistantMessage);
  }

  private filterMessages(messages: ChatMessage[], options?: { limit?: number; before?: string }): ChatMessage[] {
    if (!options) {
      return messages;
    }

    let filteredMessages = messages;

    if (options.before) {
      const beforeIndex = messages.findIndex(m => m.id === options.before);
      if (beforeIndex !== -1) {
        filteredMessages = messages.slice(0, beforeIndex);
      }
    }

    if (options.limit) {
      filteredMessages = filteredMessages.slice(-options.limit);
    }

    return filteredMessages;
  }
}
```
