1. INTRODUCTION TO INTERFACES AND DTOS
==============================================================

Interfaces and Data Transfer Objects (DTOs) are essential components of the Jibu Console backend architecture. They provide type safety, enforce contracts between different parts of the application, and facilitate data validation and transformation.

Interfaces define the structure of objects and the contracts that classes must adhere to. They are used to define entities, service contracts, and utility types. DTOs, on the other hand, are specialized interfaces used for transferring data between different layers of the application, particularly between the client and the server.

The Jibu Console backend uses TypeScript interfaces and classes extensively to ensure type safety and provide clear contracts between different components. This document provides a comprehensive overview of the key interfaces and DTOs used in the system.

==============================================================
2. DATA TRANSFER OBJECTS (DTOS)
==============================================================

Data Transfer Objects (DTOs) are plain objects used to transfer data between different layers of the application. In the Jibu Console backend, DTOs are primarily used for:

1. Validating incoming request data
2. Transforming data between the API layer and the service layer
3. Structuring response data for the client

DTOs are implemented as TypeScript classes with decorators for validation and transformation. The backend uses the class-validator and class-transformer libraries to implement validation and transformation logic.

2.1. Request DTOs
---------------

Request DTOs define the structure and validation rules for incoming request data. They are used to validate and transform data received from the client before it is processed by the service layer.

Example: CreateAgentDto

```typescript
export class CreateAgentDto {
  @IsString()
  @IsNotEmpty()
  @MinLength(3)
  @MaxLength(50)
  name: string;

  @IsString()
  @IsNotEmpty()
  description: string;

  @IsString()
  @IsNotEmpty()
  @IsUUID()
  organizationId: string;

  @IsObject()
  @ValidateNested()
  @Type(() => AgentConfigDto)
  config: AgentConfigDto;

  @IsOptional()
  @IsArray()
  @IsString({ each: true })
  @IsUUID(undefined, { each: true })
  knowledgeBaseIds?: string[];

  @IsOptional()
  @IsString()
  @IsUUID()
  credentialId?: string;

  @IsOptional()
  @IsBoolean()
  isPublic?: boolean;
}
```

2.3. Common DTOs
--------------

Common DTOs are used across multiple modules and provide shared functionality.

Example: PaginationQueryDto

```typescript
export class PaginationQueryDto {
  @IsOptional()
  @IsInt()
  @Min(1)
  @Type(() => Number)
  page?: number = 1;

  @IsOptional()
  @IsInt()
  @Min(1)
  @Max(100)
  @Type(() => Number)
  limit?: number = 10;

  @IsOptional()
  @IsObject()
  @Transform(({ value }) => {
    try {
      return typeof value === 'string' ? JSON.parse(value) : value;
    } catch {
      return {};
    }
  })
  filter?: Record<string, any> = {};

  @IsOptional()
  @IsObject()
  @Transform(({ value }) => {
    try {
      return typeof value === 'string' ? JSON.parse(value) : value;
    } catch {
      return {};
    }
  })
  sort?: Record<string, 'ASC' | 'DESC'> = {};
}
```

Example: ApiResponseDto

```typescript
export class ApiResponseDto<T> {
  @Expose()
  success: boolean;

  @Expose()
  message?: string;

  @Expose()
  data?: T;

  @Expose()
  error?: string;

  @Expose()
  timestamp: Date;

  constructor(partial: Partial<ApiResponseDto<T>>) {
    Object.assign(this, partial);
    this.timestamp = new Date();
  }

  static success<T>(data: T, message?: string): ApiResponseDto<T> {
    return new ApiResponseDto<T>({
      success: true,
      data,
      message,
    });
  }

  static error<T>(error: string, message?: string): ApiResponseDto<T> {
    return new ApiResponseDto<T>({
      success: false,
      error,
      message,
    });
  }
}
```
