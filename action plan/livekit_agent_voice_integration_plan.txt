# LIVEKIT AGENT VOICE MODE INTEGRATION PLAN

## OVERVIEW
This plan focuses specifically on integrating LiveKit voice mode with the agent execution workflow in Jibu Console. The integration will focus on both backend (AgentExecutionService) and frontend (AgentExecutionDialog) components.

✅ LiveKit client dependencies have been installed:
- livekit-client
- @livekit/components-react
- @livekit/components-styles

### 1. EXTEND AGENT EXECUTION SERVICE

1. Modify AgentExecutionService in apps/backend/src/modules/v1/agent/execution/agent-execution.service.ts:
   - Add voice session management methods:
     - initiateAgentVoiceSession(agentId, userId)
     - endAgentVoiceSession(sessionId)
     - handleVoiceStateChange(sessionId, state)
   - Add voice mode state handling:
     - pauseAgentVoiceOutput()
     - resumeAgentVoiceOutput()

2. Create voice-specific DTOs:
   - apps/backend/src/modules/v1/agent/execution/dto/voice-execution.dto.ts

### 2. CREATE AGENT VOICE SERVICES

   - Define states: IDLE, AGENT_SPEAKING, USER_SPEAKING, SILENCE, INTERRUPTION

1. Implement agent voice monitoring:
