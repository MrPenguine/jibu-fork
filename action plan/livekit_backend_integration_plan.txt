# LIVEKIT BACKEND INTEGRATION PLAN

## BACKEND DEPENDENCIES

1. Add LiveKit dependencies:
   pnpm add livekit-server-sdk

2. Additional dependencies:
   pnpm add @nestjs/websockets @nestjs/platform-socket.io

## BACKEND MODULES

### 1. LIVEKIT MODULE

1. Create directory structure:
   mkdir -p apps/backend/src/modules/v1/livekit/dto

## DATABASE SCHEMA UPDATES

1. Modify Prisma schema (apps/backend/prisma/schema.prisma) to add:
   - VoiceSession model with fields:
     - id (UUID)
     - userId (relation to User)
     - chatId (optional relation to Chat)
     - assistantId (optional relation to Assistant)
     - agentId (optional relation to Agent)
     - roomName (string)
     - status (enum: ACTIVE, ENDED)
     - createdAt (DateTime)
     - endedAt (DateTime, optional)

2. Generate Prisma client:
   cd apps/backend
   npx prisma generate

3. Create migration:
   cd apps/backend
   npx prisma migrate dev --name add_voice_session

## BACKEND VOICE FEATURES

### 1. VOICE ACTIVITY DETECTION

1. Implement WebRTC audio analysis:
   - Configure silence thresholds
   - Set up real-time audio level monitoring
   - Create state tracking for active/inactive voice

2. Implement speech event handling:
   - Create WebSocket gateway for real-time events
   - Set up Redis pub/sub for distributed events
   - Build event handlers for speech start/stop

### 2. CONVERSATION STATE MACHINE

1. Design voice conversation states:
   - IDLE: No active speakers
   - USER_SPEAKING: User is currently speaking
   - AGENT_SPEAKING: Agent/Assistant is currently speaking
   - SILENCE: No speech detected for configurable period

2. Implement state transitions:
   - Define rules for state changes
   - Create event triggers for transitions
   - Track conversation state in Redis

### 3. SILENCE DETECTION

1. Implement silence detection service:
   - Configure silence thresholds and timeouts
   - Create "Are you there?" prompt generation
   - Build middleware for handling detected silence

### 4. DTMF HANDLING

1. Implement DTMF detection:
   - Set up DTMF tone listeners in LiveKit
   - Create event handlers for keypad inputs
   - Design command structure for navigation

2. Create DTMF API endpoint:
   - POST endpoint to handle DTMF commands
   - Service methods to process commands
   - Integration with chat/agent workflows
